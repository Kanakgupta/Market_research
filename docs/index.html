Stack Primary Targets Positioning Strength Trade-off / Gap Missing Features Best Fit
Bluetooth Stack SoCs/modules, embedded MCU and Linux gateway designs Tight silicon + stack integration, low-power tuning, and vendor-backed lifecycle/support Less portable outside ecosystem than fully open host stacks No native cross-vendor portability layer for non- silicon Commercial products prioritizing time-to-market, qualification, and long-term support
Zephyr Bluetooth HostOpen Source RTOS-based embedded IoT devices across many MCU vendors Portable open-source stack, broad board support, high customization flexibility More integration/maintenance ownership for product teams No single-vendor production support SLA and less turnkey certification assistance Teams optimizing for cross-vendor firmware reuse and deep embedded customization
BlueZOpen Source Linux hosts/gateways, edge computers, and application processors Mature Linux ecosystem integration with strong host-side interoperability Not an RTOS device stack; typically needs Linux-class host footprint Limited direct fit for tiny MCU-only endpoints without a Linux host Gateway/host products where Linux is already the system baseline
Apache NimBLEOpen Source Resource-constrained MCU products and embedded RTOS environments Small memory footprint and widely reused BLE host for constrained systems Feature integration breadth depends on chosen platform/vendor SDK No unified full-stack toolchain across all MCU vendors Ultra-low-resource BLE endpoints prioritizing compact implementation
BTstackOpen Source Embedded devices, prototyping platforms, and custom Bluetooth firmware Clean architecture and flexibility for custom protocol/application work Requires stronger in-house Bluetooth expertise for production hardening Smaller ecosystem and fewer out-of-box vertical integrations than mainstream stacks Engineering-led products needing highly tailored stack behavior
Android Fluoride StackOpen Source Android phones/tablets and Android-based consumer devices Deep integration with Android framework and large deployed device base Tightly coupled to Android platform release and vendor customization layers Not suitable as a drop-in RTOS stack for standalone MCU peripherals Products with Android as the primary OS and app-facing Bluetooth requirements

Bluetooth

Now: Bluetooth Core Specification 6.3 (adopted) Next: Not yet announced (6.3 is the latest adopted release)
  • Channel Sounding Inline PCT Transfer (BT 6.3) — analog phase pre-compensation for faster, lower-overhead ranging
  • Channel Sounding PHY-specific RTT Accuracy (BT 6.3) — per-PHY ranging precision (LE 1M/2M)
  • Running Out of Bits (BT 6.3) — expanded HCI command/event bitmask for future feature growth

Platform CYW20829 ships Channel Sounding (PBR/RTT baseline from BT 6.0); CYW55513 covers LE Audio. Track BT 6.3 Channel Sounding refinements (Inline PCT Transfer, per-PHY RTT) for next silicon revs.
